## 2. How an Agent Thinks: The Reasoning Loop

Regular AI is a straight line: you ask, it answers. One shot. Done.

An agentic AI is different. It **thinks in circles** — planning what to do, doing it, checking the results, and adjusting if something's off. Then it loops again. This is called the **ReAct pattern** (Reasoning + Acting).

<ReasoningLoop />

Watch the dot orbit through the four phases. Each pass makes the answer better. If something doesn't add up, the agent doesn't guess — it **goes back and fixes it**.

## This Pattern Works Everywhere

The reasoning loop isn't tied to one use case. Change the tools and the rules, and you can apply it to:

- **Finance** — checking transactions against policies
- **Healthcare** — verifying data flows against privacy rules
- **Environment** — comparing sensor readings to safety limits
- **Construction** — reviewing plans against building codes

The loop is the pattern. The domain is just configuration.

## Try It Yourself

1. **Pick an orchestrator** — LangGraph is a good start
2. **Load your rules** into a searchable database (vector DB works great)
3. **Build 2–3 small tools** the agent can call (checkers, lookups, validators)
4. **Add the guardrail**: the agent must cite a rule before flagging anything
5. **Log everything** — the trail of decisions is the real product

The best system isn't one that's always right. It's one that can **show its work**.
